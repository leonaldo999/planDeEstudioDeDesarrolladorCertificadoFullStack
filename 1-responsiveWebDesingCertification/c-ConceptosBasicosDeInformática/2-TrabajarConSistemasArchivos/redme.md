# Trabajar con sistemas de archivos

En estos videos de conferencias, aprenderá a trabajar con sistemas de archivos y carpetas en sus computadoras. Aprenderá a crear, mover y eliminar archivos y carpetas, las mejores prácticas para nombrar y organizar archivos y carpetas, y más.

---

## ¿Cómo utilizar las aplicaciones de gestión de archivos?

Una `aplicación de administración de archivos` es una forma de almacenar, organizar y editar fácilmente sus archivos en su computadora. Ya sea que use una computadora `Mac` o `Windows`, hay aplicaciones integradas que puede usar para organizar sus archivos.

Si tiene una computadora con `Windows`, el administrador de archivos predeterminado es el `Explorador de archivos`. Este se utiliza para explorar, buscar y administrar archivos y carpetas, así como para realizar operaciones con archivos como copiar, mover y eliminar.

Para encontrar el `Explorador de archivos`, puede ir al menú Inicio o presionar `la tecla del logotipo de Windows` en su teclado.

`Para anclar una carpeta`, haga clic derecho y seleccione `"Anclar"`. Para mover un archivo o carpeta, primero selecciónelo, luego seleccione `"Cortar"` y navegue hasta la nueva ubicación para pegarlo.

Si tienes una computadora `Mac`, el administrador de archivos predeterminado es el Finder. Este te permite acceder a archivos locales y remotos, previsualizar archivos y ofrece diversas funciones de organización, como etiquetas y carpetas inteligentes.

Para acceder al Finder en tu `Mac`, ve al Dock en la parte inferior y haz clic en el ícono del Finder.

Notarás una barra lateral en el lado izquierdo donde se encuentran tus carpetas favoritas.

Una forma de organizar tus archivos es usar etiquetas. Para etiquetar un archivo o carpeta, selecciónalo en el Finder y haz clic derecho. Verás la opción "Etiquetas..." en el menú desplegable y podrás aplicar una etiqueta de color personalizada.

Para buscar elementos etiquetados, puede desplazarse hasta la parte inferior de la barra lateral y hacer clic en la etiqueta de color correspondiente, o puede buscar la etiqueta por nombre de color en el campo de búsqueda.

Otra forma de organizar tus archivos es usar carpetas inteligentes. Estas carpetas son conjuntos de archivos que se organizan automáticamente según criterios que establezcas, como el tipo de archivo, la fecha o las palabras clave.

Ya sea que use el Explorador de archivos en Windows o el Finder en Mac, estas aplicaciones brindan flexibilidad para administrar sus archivos con funciones como etiquetas, búsqueda y carpetas inteligentes.

## ¿Cuáles son las mejores prácticas para nombrar archivos para aplicaciones web?

A medida que comience a crear sus propias aplicaciones web, deberá tener cuidado con el nombre que le da a sus archivos.

Querrá nombrar sus archivos de una manera que sea fácil de entender y mantener.

Veamos algunos ejemplos de nombres de archivo HTML adecuados e inadecuados. Tenga en cuenta que la `.html` extensión en estos ejemplos indica que se trata de un archivo HTML.

*He aquí un ejemplo de un nombre de archivo incorrecto:*

- **Codigo ejm:**

  ```html
  index1234.html
  ```

Solo por el nombre, no queda claro de qué se trata el archivo. Podría ser para una página de productos, una página de blog o cualquier otro tipo de página, pero no lo sabemos por el nombre.

*A continuación se muestra un ejemplo de un mejor nombre de archivo para un archivo HTML:*

- **Codigo ejm:**

  ```html
  about-us.html
  ```

Al utilizar un nombre más descriptivo como about-us.html, queda claro de qué trata el archivo.

También notarás que en este nombre de archivo no hay espacios. En su lugar, se usa un guion, -, para separar las palabras.

A veces, es posible que veas un nombre de archivo llamado index.html. Este es un nombre de archivo especial que representa la página principal de un sitio web. Al visitar un sitio web, este index.htmlarchivo es el primero que se carga.

Otra consideración importante al nombrar archivos es el uso de caracteres especiales.

El uso de una combinación de caracteres especiales, números y letras puede dificultar la comprensión de qué trata el archivo.

*He aquí un ejemplo de un nombre de archivo incorrecto:*

- **Codigo ejm:**

  ```html
  file-1!@.html
  ```

Este nombre parece innecesariamente complicado y no nos brinda ninguna información sobre de qué se trata el archivo.

Por lo tanto, es mejor utilizar únicamente letras y guiones en los nombres de los archivos.

A medida que comience a aprender más idiomas, verá que también existen convenciones comunes para nombrar archivos en esos idiomas.

En estas situaciones, siempre es mejor consultar la documentación oficial u otros recursos para comprender las mejores prácticas para nombrar archivos en ese idioma.

También estarás en situaciones en las que trabajarás en equipo y necesitarás seguir las convenciones que el equipo haya establecido para nombrar archivos.

Consulte siempre la guía de estilo del equipo para asegurarse de que los nombres de sus archivos sean coherentes con el resto del equipo.

## ¿Cuáles Son las Mejores Prácticas para la Organización de Archivos/Carpetas en Aplicaciones Web?

Al crear aplicaciones web, es importante pensar en la organización de sus archivos y carpetas. Esto ayudará a mantener su código organizado y más fácil de mantener.

Echemos un vistazo a una estructura de carpetas de ejemplo para un proyecto `HTML` y `CSS`:

```txt
.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md
```

En la parte superior del ejemplo, hay un solo punto, que representa el directorio raíz.

El directorio raíz es el directorio de nivel superior en un sistema de archivos, que sirve como punto de partida para todas las rutas de archivos y contiene todos los demás directorios y archivos. Un directorio es otro nombre para una carpeta.

Dentro del directorio raíz, el primer directorio que se muestra es el directorio `assets`. El término `assets` se refiere a cualquier archivo que se utilice en el proyecto, como imágenes, fuentes u otros recursos.

En este ejemplo, hay dos directorios dentro del directorio `assets` llamados `images` y `fonts`.

El siguiente directorio sería el directorio `css`. A veces, esto también se conoce como directorio de hojas de estilo. Aquí es donde almacenarías todos tus archivos CSS.

Debajo del directorio `css` están los archivos HTML. El archivo `index.html` representa la página de inicio, mientras que `about.html` y `contact.html` representa las páginas `About` y `Contact`, respectivamente.

Por último, hay un archivo `README.md`. Los archivos `README` se usan comúnmente para proporcionar información útil sobre el proyecto, como qué hace, cómo usarlo y cualquier otro detalle relevante.

La extensión `.md` significa `Markdown`, que es un lenguaje de marcado liviano que a menudo se usa para escribir documentación.

Al organizar su proyecto en activos, `CSS` y archivos `HTML`, facilita la búsqueda de lo que está buscando. Esto también facilita que otros desarrolladores contribuyan a su proyecto.

Es importante tener en cuenta que este es solo un ejemplo de cómo puede organizar sus archivos y carpetas. Hay muchas formas diferentes de estructurar un proyecto.

## ¿Cómo puedes crear, mover, y eliminar archivos y carpetas usando el Explorador de Windows o el Finder de macOS?

Para crear un archivo en Windows con el Explorador de archivos, haga clic en la opción `"Nuevo"` en la esquina superior derecha. Esto le mostrará una lista de los diferentes tipos de archivos que puede crear.

También puedes crear archivos en sus aplicaciones correspondientes. Crearemos un archivo de texto para mostrarte el proceso.

Al hacer clic en `"Nuevo"` y seleccionar el tipo de archivo, verá un nuevo archivo de texto en su carpeta actual con un nombre predeterminado. Puede cambiarle el nombre si lo desea.

El proceso para crear carpetas es exactamente el mismo. Haz clic en el botón `"Nuevo"` en la esquina superior derecha y selecciona la opción `"Carpeta"` o haz clic derecho en un espacio vacío y ve a `"Nuevo"`. Luego, selecciona la opción `"Carpeta"`. Verás una nueva carpeta vacía en tu ubicación actual.

Puede renombrar archivos y carpetas seleccionándolos y haciendo clic en la opción `"Cambiar nombre"` en la parte superior. Cuando el nombre actual del archivo esté resaltado, puede empezar a escribir el nuevo nombre. Pulse Intro para confirmar los cambios.

También puede hacer clic derecho en el archivo o carpeta y seleccionar la opción `"Cambiar nombre"` o usar la tecla de acceso directo `F2`. Escriba el nuevo nombre y presione Intro para confirmar los cambios.

Puedes mover un archivo o carpeta a otra carpeta arrastrándolo y soltándolo en la carpeta de destino, si la tienes abierta como una pestaña.

También puedes seleccionar el archivo o carpeta y seleccionar la opción `“Cortar”` en la parte superior.

Luego, vaya a la carpeta de destino y péguelo haciendo clic en el botón `"Pegar"` en la parte superior. Si copió o cortó el archivo previamente, este botón estará habilitado.

También puede hacer esto haciendo clic derecho en un espacio vacío y luego haciendo clic en el icono de pegar. Esto eliminará el archivo o la carpeta de su ubicación original.

Para eliminar un archivo o carpeta, haz clic en él y luego en el icono de la Papelera en la parte superior. También encontrarás la opción `"Eliminar"` si haces clic derecho sobre él.

Genial. Ahora veamos cómo puedes hacer esto en macOS usando el Finder.

Finder es el administrador de archivos predeterminado en macOS.

Para crear un archivo en macOS, necesitas abrir una aplicación que te permita crear el tipo de documento que necesitas. Actualmente, no existe una forma integrada de crear un archivo directamente en el Finder.

Por ejemplo, para crear un archivo de texto simple, puede abrir TextEdit y guardarlo. El proceso funciona igual para cualquier tipo de archivo.

Hay muchas maneras de abrir TextEdit. Una de ellas es ir al Launchpad. Luego, busca una aplicación que te permita crear el tipo de archivo que te interesa. En este caso, sería `"TextEdit"`. Haz clic en el icono para abrir la aplicación.

Una vez dentro de la app, deberías ver una opción para guardar tu archivo en el menú de la app. Este menú se encuentra en la esquina superior derecha, junto al icono del menú de Apple.

En una de las opciones del menú de la aplicación, deberías ver un comando para guardar el archivo. Normalmente se encuentra en el menú Archivo.

Junto a él, verás el atajo de teclado para guardar el archivo. Suele estar disponible `Command + S` en macOS.

Cuando guarde el archivo por primera vez, debería ver una ventana de diálogo, donde puede escribir el nombre del archivo y elegir una ubicación.

Generalmente, la extensión del archivo la agrega la aplicación automáticamente, pero puedes cambiarla antes de guardar el archivo.

Después de guardar el archivo, puede abrir el Finder haciendo clic en su icono en el Dock. Debería ver el nuevo archivo en la carpeta donde lo guardó.

Para crear una carpeta nueva, vaya a la ubicación donde desea crearla y haga clic con el ratón en un espacio vacío. Verá una lista de opciones. La primera es `"Nueva carpeta"`.

Si hace clic en él, puede asignarle un nombre a su nueva carpeta. Escriba el nombre y presione Enter para confirmar.

Hay varias maneras de mover archivos y carpetas en el Finder. Si necesitas moverlos a una carpeta de la sección `"Favoritos"` del Finder, simplemente arrástrala y suéltala.

Si necesita moverlo a una carpeta diferente, tiene dos opciones.

Puede abrir la carpeta de destino como una pestaña y arrastrar y soltar el archivo o la carpeta en la pestaña.

También puedes usar atajos de teclado. Primero, copia el archivo con `Command + C`, ve a la nueva carpeta y luego usa `Command + Option + V` para pegar el archivo o la carpeta en esa ubicación. Esto también lo eliminará de la carpeta original.

Para eliminar un archivo o carpeta, puede hacer clic derecho sobre él y seleccionar `"Mover a la papelera"`.

También puedes arrastrarlos y soltarlos en el icono de la papelera del Dock. Esto es equivalente.

Cuando el archivo o carpeta esté en la papelera, puede hacer clic derecho sobre él para eliminarlo inmediatamente o puede vaciar la papelera para eliminar todos los archivos y carpetas eliminados de forma permanente.

Saber cómo crear, mover y eliminar archivos y carpetas en Windows y macOS es fundamental. Al dominar estas habilidades, podrá organizar sus archivos, localizarlos rápidamente y liberar espacio en su computadora eliminando archivos innecesarios.

## ¿Cómo puedes buscar archivos y carpetas en tu computadora?

Aprendamos a buscar archivos y carpetas en **Windows** y **macOS**.

**Empecemos con Windows**. Puedes buscar desde la barra de tareas o desde el Explorador de archivos.

Para buscar desde la barra de tareas, escribe el nombre del archivo o carpeta que buscas. También puedes buscar por palabras clave. Verás resultados de tu PC y OneDrive.

Si solo quieres ver documentos, puedes hacer clic en la pestaña "Documentos". También tienes pestañas para aplicaciones y resultados web.

Otra alternativa es usar el Explorador de archivos de búsqueda. Lo encontrarás en la esquina superior derecha de la ventana del Explorador, donde verás "Buscar documentos".

*Tiene tres opciones para definir el alcance de la búsqueda:*

- Puedes buscar desde Inicio(`Home`) para encontrar archivos desde tu PC y desde la nube rápidamente.

- Puedes buscar en una carpeta para encontrar archivos almacenados dentro de ella.

- Puedes buscar desde Esta PC para realizar una búsqueda lenta pero profunda.

Así es como puedes buscar archivos y carpetas en Windows.

Ahora veamos cómo hacerlo en macOS.

Primero, puedes usar Spotlight, una herramienta que te ayuda a encontrar archivos en tu Mac. Para abrir Spotlight, ve a la barra de menú en la esquina superior derecha y busca el icono de la lupa. Si no lo ves en la barra de menú, puedes añadirlo en la configuración del Centro de control.

Ve a tu configuración, filtra por "Centro de control", haz clic en esta opción y desplázate hacia abajo hasta encontrar "Spotlight". También puedes usar un atajo de teclado para abrir Spotlight `Command + Spacebar`:

Al abrir Spotlight, verá la Búsqueda de Spotlight, donde puede buscar cualquier archivo o carpeta escribiendo su nombre. Al empezar a escribir, verá los resultados agrupados por categoría, incluyendo sugerencias, carpetas, presentaciones, fotos, documentos y búsquedas relacionadas.

También puede expandir una categoría específica haciendo clic en "Mostrar más".

Otra forma de buscar archivos que has abierto recientemente es abrir el Finder haciendo clic en su icono en el Dock. Luego, ve a "Recientes" en la barra lateral del Finder. Allí encontrarás todos los archivos que has visto recientemente.

Así puedes buscar archivos y carpetas en macOS. Con estas herramientas de búsqueda, puedes optimizar tu flujo de trabajo y encontrar rápidamente los archivos y carpetas que necesitas.

## ¿Cuáles son algunos tipos de archivos comunes con los que trabajarás en aplicaciones web?

Los archivos se clasifican según su contenido y estructura. El tipo de archivo determina cómo los programas informáticos lo abren y lo procesan.

Identificamos los tipos de archivo según sus extensiones. Una extensión de archivo es un sufijo de tres o cuatro letras al final del nombre del archivo. Los tres tipos de archivo principales que encontrará en las aplicaciones web son HTML, CSS y JavaScript.

Los archivos HTML contienen la estructura y el contenido de la aplicación web. Tienen una `.html` extensión de archivo.

Los archivos CSS definen los estilos. Tienen una `.css` extensión de archivo.

Los archivos JavaScript añaden funcionalidad e interactividad más avanzadas. Tienen una `.js` extensión de archivo.

Al desarrollar aplicaciones web, también necesitará incluir imágenes. Estos son algunos de los tipos de archivo de imagen más utilizados.

JPEG, acrónimo de "Joint Photographic Experts Group", es conocido por su eficiente compresión. Es ideal para fotografías e imágenes. Sin embargo, la compresión presenta pérdidas, lo que significa que se pierde parte de la calidad de la imagen para reducir el tamaño del archivo.

PNG, que significa "Gráficos de Red Portátiles", conserva la calidad de la imagen incluso después de la compresión. Esto resulta en archivos más grandes. PNG admite transparencia y es ideal para imágenes con bordes nítidos, como logotipos e iconos.

El formato de imagen GIF admite animación y transparencia, pero tiene una paleta de colores limitada. GIF significa "Formato de Intercambio de Gráficos".

Y finalmente, tenemos SVG, que significa "Gráficos Vectoriales Escalables". Es un formato de imagen vectorial. Estas imágenes se pueden escalar según sea necesario sin perder calidad.

Genial. Ahora que ya sabes sobre formatos de imagen, veamos algunos formatos de video y audio.

MP3 es un formato de audio conocido por su eficiente compresión. Es un formato con pérdida de datos, por lo que se pierden algunos datos de audio durante la compresión para reducir el tamaño de los archivos.

MP4 es uno de los formatos de vídeo más comunes. Ofrece buena compresión y admite múltiples códecs de audio y vídeo, subtítulos y metadatos.

MOV, el formato de archivo multimedia QuickTime desarrollado por Apple, se asocia principalmente con QuickTime Player.

Otros tipos de archivos de audio y vídeo populares son WAV, un formato de audio sin pérdida que mantiene la calidad original del audio, y WebM, un formato de vídeo de código abierto de alta calidad.

Si alguna vez necesitas personalizar las fuentes de tu aplicación web, también trabajarás con formatos de fuente. Estos son tres de los más utilizados.

TTF, que significa “TrueType Font”, es un formato de fuente compatible con diferentes sistemas operativos.

WOFF es un formato de fuente moderno, diseñado específicamente para el desarrollo web. Significa "Web Open Font Format" (Formato de Fuente Web Abierta). Estos archivos son más pequeños gracias a su mejor compresión. También pueden almacenar metadatos, incluyendo información de licencias.

Y WOFF2 es el sucesor de WOFF con una compresión y un rendimiento aún más eficientes.

También puedes crear archivos comprimidos si necesitas agrupar varias carpetas y archivos. El formato de archivo más utilizado es ZIP. ZIP ofrece compresión sin pérdidas, por lo que no se pierden datos durante el proceso. Es ampliamente compatible con sistemas operativos y aplicaciones de software.

Y, por último, hablemos de la documentación. En las aplicaciones web, normalmente encontrarás un archivo llamado `README` que contiene información sobre la aplicación, como cómo usarla, cómo instalarla, su licencia y cómo contribuir.

Suelen estar escritos en un formato llamado Markdown. Con Markdown, puedes crear documentos estructurados con encabezados, subencabezados, enlaces, imágenes, listas y más. Los archivos Markdown tienen la `.md` extensión `.markdown`.

Aquí puedes encontrar el archivos `README` del repositorio de GitHub de freeCodeCamp: `https://github.com/freeCodeCamp/freeCodeCamp/blob/main/README.md`

Puede crear esta estructura y formato detallados utilizando Markdown.

Seguramente leerás y escribirás muchos archivos `README` a lo largo de tu carrera. Aprender sobre estos tipos de archivos comunes es esencial para el desarrollo web.

Desde los componentes básicos de HTML, CSS y JavaScript hasta los formatos de imagen, video y audio, cada tipo de archivo tiene un papel importante en la creación de aplicaciones web interactivas.
