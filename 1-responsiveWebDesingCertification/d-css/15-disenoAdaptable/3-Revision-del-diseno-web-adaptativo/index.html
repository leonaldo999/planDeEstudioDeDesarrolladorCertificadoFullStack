🚀 Introducción

El diseño web adaptativo es un enfoque de diseño que se centra en crear sitios web que se adapten a diferentes tamaños de pantalla y dispositivos. Utiliza técnicas como cuadrículas fluidas, imágenes flexibles y consultas de medios para garantizar que el contenido se muestre de manera óptima en cualquier dispositivo.

Diseño web adaptativo

  • Definición: El principio fundamental del diseño adaptativo es la adaptabilidad, es decir, la capacidad de un sitio web para ajustar su diseño y contenido en función del tamaño de la pantalla y las capacidades del dispositivo en el que se visualiza. 
/* DEFINICIÓN: ADAPTABILIDAD */
.adaptive-container {
  width: 100%;
  transition: all 0.3s ease;
}

@media (min-width: 768px) {
  .adaptive-container {
    width: 80%;
    margin: 0 auto;
  }
}
Contenido Adaptativo

💡 Revisión: La adaptabilidad es la capacidad de ajustar el layout y el contenido según el dispositivo.

  • Grids fluidas: utilizan unidades relativas, como porcentajes, en lugar de unidades fijas, como píxeles, lo que permite que el contenido cambie de tamaño y se reajuste en función del tamaño de la pantalla. 
/* GRIDS FLUIDAS: Unidad relativa (porcentaje) */
.column-main {
  width: 70%; /* Flexible respecto al padre */
  float: left;
}

.column-sidebar {
  width: 30%;
  float: left;
}
70%
30%

💡 Revisión: Al usar porcentajes, si el contenedor padre cambia de tamaño, las columnas se reajustan proporcionalmente de forma automática.

  • Imágenes flexibles: están configuradas para cambiar de tamaño dentro de los elementos que las contienen, lo que garantiza que no se salgan de sus contenedores en pantallas más pequeñas. 
/* IMÁGENES FLEXIBLES: Control total */
img {
  max-width: 100%; /* Nunca supera su contenedor */
  height: auto;     /* Mantiene proporción original */
  display: block;   /* Elimina espacios inferiores */
}
Imagen Flexible (100%)

💡 Revisión: Al usar max-width: 100%, le decimos a la imagen: "puedes ser pequeña si tu contenedor lo es, pero nunca te hagas más grande que tu tamaño real".

Consultas de @media queries

  • Definición: permiten a los desarrolladores aplicar diferentes estilos en función de las características del dispositivo, principalmente el ancho de la ventana gráfica. 
<p class="mq-example">
  The background changes at 768px.
</p>
.mq-example {
  padding: 10px;
  background-color: lightyellow;
}

/* Breakpoint en 768px */
@media screen and (min-width: 768px) {
  .mq-example {
    background-color: lightblue;
  }
}

The background of this paragraph will change when the viewport is at least 768px wide.

💡 Revisión: El navegador reevalúa las media queries en tiempo real. ¡No necesitas recargar la página para ver cómo tu diseño se transforma!

  • Tipos de medios de all: adecuado para todos los dispositivos. Es el valor predeterminado si no se especifica ningún tipo de medio. 
/* TIPO DE MEDIO: all (Global) */

/* Estas reglas se aplican
   a TODOS los dispositivos */
@media all {
  body {
    font-family: sans-serif;
  }
}

/* Equivalente a escribir
   solo el selector sin @media */
body {
  font-family: sans-serif;
}
Nota: @media all es la configuración base. Se usa para estilos que no deben cambiar nunca, sin importar dónde se vea la web.
  • Tipos de medios de print: destinado a material paginado y documentos que se visualizan en una pantalla en modo de vista previa de print. 
/* TIPO DE MEDIO: print */

@media print {
  nav, .sidebar, .ads {
    display: none; /* Ocultar elementos innecesarios */
  }

  body {
    color: black;
    font-size: 12pt; /* Ajuste para impresión */
  }
}
Nota: Al usar @media print, puedes definir una hoja de estilos limpia, ideal para que el lector solo obtenga el contenido relevante.

💡 Revisión: Esta configuración es la que activa el navegador cuando el usuario selecciona "Archivo > Imprimir" o presiona Ctrl+P.

  • Tipos de medios de screen: destinado principalmente a screens. 
/* TIPO DE MEDIO: screen */

@media screen {
  /* Solo se aplica en dispositivos
     con pantalla */
  body {
    background-color: #f4f4f4;
  }
}

/* Combinado con otras condiciones */
@media screen and (min-width: 1024px) {
  .sidebar {
    display: block;
  }
}
Nota: @media screen es esencial para separar estilos de pantalla de los estilos de impresión (print) o de síntesis de voz (speech).

💡 Revisión: Aunque la mayoría de los estilos hoy en día están pensados para pantallas, declarar explícitamente screen es una buena práctica para mantener tu código semánticamente correcto.

  • aspect-ratio: describe la relación entre la anchura y la altura de la ventana gráfica. 
/* ASPECT-RATIO: Relación ancho/alto */

/* Pantallas panorámicas (16:9 o más) */
@media (min-aspect-ratio: 16/9) {
  body { background-color: #f0f8ff; }
}

/* Pantallas verticales (portrait) */
@media (max-aspect-ratio: 1/1) {
  body { background-color: #fff0f0; }
}
Nota: aspect-ratio es ideal para evitar que el contenido se vea "aplastado" en formatos de pantalla inusuales.

💡 Revisión: Esta propiedad nos permite ir un paso más allá del simple ancho, enfocándonos en la geometría del espacio disponible.

  • Orientation: se utiliza para indicar si el dispositivo está en orientación horizontal o vertical. 
/* ORIENTATION: Vertical o Horizontal */

/* Vertical (Portrait) */
@media (orientation: portrait) {
  .sidebar { display: none; }
}

/* Horizontal (Landscape) */
@media (orientation: landscape) {
  .sidebar { display: block; }
}
Nota: orientation ayuda a optimizar el espacio útil. En portrait, priorizamos el contenido vertical; en landscape, aprovechamos el ancho extra.

💡 Revisión: Esta propiedad es fundamental para que el usuario no tenga que hacer scroll infinito en dispositivos móviles cuando los gira.

  • resolution: Se utiliza para describir la resolución del dispositivo de salida en puntos por pulgada (ppp) o puntos por centímetro (ppc). 
/* RESOLUTION: Densidad de píxeles */

/* Alta densidad de píxeles (Retina, 4K, etc.) */
@media (min-resolution: 300dpi) {
  .logo {
    background-image: url('logo-hd.png');
  }
}

/* Densidad estándar */
@media (min-resolution: 72dpi) and (max-resolution: 299dpi) {
  .logo {
    background-image: url('logo-std.png');
  }
}
Nota: Usar resolution es clave para optimizar recursos. Evita enviar imágenes de alta resolución a dispositivos que no tienen la densidad de píxeles para mostrar el detalle extra.

💡 Revisión: Aunque solemos usar min-resolution, también puedes usar dppx (dots per pixel unit), que es la unidad estándar actual para manejar resoluciones de pantalla.

  • hover: Se utiliza para comprobar si el mecanismo de entrada principal puede situarse sobre los elementos. 
<button class="interactive-btn">Hover over me</button>
.interactive-btn {
  padding: 10px 20px;
  background-color: lightgray;
}

/* 
  Solo aplicar hover en dispositivos con cursor preciso
*/
@media (hover: hover) {
  .interactive-btn:hover {
    background-color: yellow;
  }
}

💡 Revisión: Esta técnica es una excelente práctica de diseño adaptable, ya que evita que los estados de :hover se "peguen" o se comporten erráticamente en dispositivos móviles (donde el toque es el evento principal).

  • prefers-color-scheme: Se utiliza para detectar si el usuario ha solicitado un tema de colores claros u oscuros. 
<div class="theme-box">
  Contenido adaptativo según sistema.
</div>
/* Colores por defecto (Light Mode) */
.theme-box {
  background-color: #ffffff;
  color: #000000;
}

/* Detección de tema oscuro */
@media (prefers-color-scheme: dark) {
  .theme-box {
    background-color: #121212;
    color: #ffffff;
  }
}
Contenido adaptativo según sistema.

💡 Revisión: Esta media query es increíble porque respeta la configuración que el usuario ya eligió en su sistema operativo, brindando una experiencia mucho más natural.

  • Consultas de medios y operadores lógicos: El operador and se utiliza para combinar varias características de medios, mientras que not y only solo se pueden utilizar para negar o aislar consultas de medios. 
<p class="logical-example">
  Cambio condicional complejo.
</p>
/* Operadores: and, not, only */
@media screen and (min-width: 768px) and (orientation: landscape) {
  .logical-example {
    background-color: lightcoral;
  }
}

/* Ejemplo de negación */
@media not print {
  /* Solo para dispositivos que NO sean impresión */
}
This paragraph's background will change when the screen is at least 768px wide AND in landscape orientation.

💡 Revisión: El operador only se usa para ocultar estilos de navegadores antiguos que no soportan media queries. Es una capa extra de seguridad para tu código.

Breakpoints comunes en los @media

  • Definición: Los Breakpoints en los media son puntos específicos en el diseño de un sitio web en los que el diseño y el contenido se ajustan para adaptarse a diferentes tamaños de pantalla. Existen algunos Breakpoints generales que se pueden utilizar para adaptarse a teléfonos, tabletas y pantallas de computadoras de escritorio. Sin embargo, no es aconsejable intentar abarcar todos los tamaños de pantalla posibles para los diferentes dispositivos. 
<p class="breakpoint-example">
  This text changes at 768px.
</p>
.breakpoint-example {
  font-size: 1rem;
  padding: 10px;
}

/* Breakpoint: Tablets y superior */
@media screen and (min-width: 768px) {
  .breakpoint-example {
    font-size: 1.125rem;
    background-color: lightblue;
  }
}

This text will change size when the viewport width is at least 768px.

💡 Revisión: Recuerda: no busques el "pixel perfecto" para cada modelo de dispositivo; busca los puntos donde tu contenido deja de verse cómodo y ahí, aplica tu breakpoint.

  • Dispositivos pequeños (smartphones): hasta 640 píxeles
  • Dispositivos medianos (tabletas): de 641 a 1024 píxeles
  • Dispositivos grandes (computadoras de escritorio): 1025 píxeles y más

Enfoque mobile first

  • Definición: El enfoque mobile first es una filosofía de diseño y una estrategia de desarrollo en el diseño web adaptativo que da prioridad a la creación de sitios web para dispositivos móviles antes que al diseño para pantallas más grandes. 
<div class="mobile-first-container">
  Diseño escalable.
</div>
/* Estilos base: Mobile (default) */
.mobile-first-container {
  width: 100%;
  background-color: lightblue;
}

/* Tablet: Escalando hacia arriba */
@media screen and (min-width: 768px) {
  .mobile-first-container { width: 750px; }
}

/* Desktop: Pantallas grandes */
@media screen and (min-width: 1024px) {
  .mobile-first-container { width: 960px; }
}
The width of this container changes based on the viewport size.

💡 Revisión: El código es más limpio porque no tienes que "sobreescribir" estilos complejos de desktop para móviles; simplemente construyes sobre una base sencilla.
