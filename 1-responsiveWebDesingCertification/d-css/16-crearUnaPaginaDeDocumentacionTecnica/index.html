Introducción a HTML

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web.

Define la estructura y el contenido de una página web mediante una serie de elementos.

<!DOCTYPE html> <html lang="es">

Cada documento HTML comienza con una declaración <!DOCTYPE html> para definir el tipo de documento.

El elemento raíz es <html>, que envuelve todo el contenido de la página.

  • Etiquetas de encabezado: <h1> a <h6>
  • Etiquetas de párrafo: <p>
  • Etiquetas de enlace: <a>
  • Etiquetas de imagen: <img>
  • Etiquetas de lista: <ul>, <ol>, <li>

Los atributos se utilizan para proporcionar información adicional sobre un elemento HTML.

Por ejemplo, el atributo src en la etiqueta <img> especifica la fuente de la imagen.

Es fundamental comprender la semántica de HTML para construir páginas accesibles y optimizadas para SEO.

La semántica se refiere al significado de los elementos.

HTML5 introdujo nuevos elementos semánticos como <header>, <nav>, <main>, <article>, <section>, <footer>.

Estos elementos ayudan a los navegadores y a las tecnologías de asistencia a entender mejor la estructura del contenido.

CSS: Fundamentos

CSS (Cascading Style Sheets) es un lenguaje de hojas de estilo utilizado para describir la presentación de un documento escrito en HTML.

Controla los colores, las fuentes, el diseño y otros aspectos visuales de una página web.

p { color: blue; } .clase { font-size: 16px; }

Existen tres formas principales de incluir CSS en un documento HTML: en línea, interno y externo.

El CSS externo es el más recomendado para proyectos grandes, ya que permite la reutilización de estilos.

  • Selectores de tipo (ej. p)
  • Selectores de clase (ej. .mi-clase)
  • Selectores de ID (ej. #mi-id)
  • Selectores de atributo (ej. [type="text"])

La cascada en CSS determina qué regla de estilo se aplica cuando hay reglas conflictivas.

La especificidad, el orden y la importancia son factores clave en la cascada.

El modelo de caja de CSS describe cómo los elementos se visualizan en la página.

Incluye contenido, padding, borde y margen.

Comprender el modelo de caja es crucial para el diseño y la maquetación.

JavaScript Esencial

JavaScript es un lenguaje de programación de alto nivel, interpretado y multiparadigma, conocido por su versatilidad.

Permite crear interactividad en las páginas web, desde animaciones hasta formularios dinámicos.

console.log("¡Hola, mundo!"); function saludar() { /* ... */ }

Las variables en JavaScript se declaran con var, let o const.

let y const fueron introducidos en ES6 y ofrecen un mejor control sobre el alcance.

  • Tipos de datos: string, number, boolean, null, undefined, object, symbol, bigint.
  • Operadores: aritméticos, de comparación, lógicos, de asignación.
  • Estructuras de control: if/else, switch, for, while.

El DOM (Document Object Model) es una interfaz de programación para documentos HTML y XML.

Permite que JavaScript acceda y manipule el contenido y la estructura de una página web.

Los eventos son acciones que ocurren en el navegador (ej. clic de un botón, envío de un formulario).

JavaScript permite "escuchar" estos eventos y ejecutar funciones en respuesta.

La programación asíncrona es fundamental para operaciones que toman tiempo, como las solicitudes de red.

Se utiliza callbacks, Promises y async/await para manejarla.

Frameworks y Librerías

Los frameworks y librerías de JavaScript simplifican y aceleran el desarrollo de aplicaciones web complejas.

Proporcionan herramientas, patrones y componentes predefinidos.

import React from 'react'; Vue.createApp({});

React: Una librería declarativa y eficiente para construir interfaces de usuario.

Se basa en componentes y un Virtual DOM para optimizar el rendimiento.

  • Componentes funcionales y de clase
  • Hooks (useState, useEffect, useContext)
  • Manejo de estado (Context API, Redux)

Angular: Un framework completo y robusto mantenido por Google.

Ofrece una estructura MVC (Model-View-Controller) y una gran cantidad de características integradas.

Vue.js: Un framework progresivo y versátil, conocido por su facilidad de aprendizaje.

Puede ser adoptado incrementalmente en proyectos existentes.

Buenas Prácticas

Adoptar buenas prácticas de desarrollo web es crucial para crear aplicaciones robustas, mantenibles y escalables.

Esto incluye la legibilidad del código, el rendimiento y la seguridad.

// Comentarios claros y concisos const MAX_ITEMS = 10;

Organización del código: Mantener una estructura de archivos lógica y modular.

Utilizar convenciones de nomenclatura consistentes.

  • Separación de preocupaciones (HTML para estructura, CSS para estilo, JS para interactividad)
  • Uso de control de versiones (Git)
  • Pruebas unitarias y de integración

Optimización del rendimiento: Minimizar y comprimir archivos (HTML, CSS, JS).

Optimizar imágenes y usar el lazy loading.

Accesibilidad (A11y): Diseñar sitios web que sean utilizables por todas las personas, incluyendo aquellas con discapacidades.

Utilizar etiquetas semánticas, texto alternativo para imágenes y contraste de colores adecuado.

Seguridad: Proteger contra vulnerabilidades comunes como inyección SQL y XSS.

Validar la entrada del usuario y usar HTTPS.
