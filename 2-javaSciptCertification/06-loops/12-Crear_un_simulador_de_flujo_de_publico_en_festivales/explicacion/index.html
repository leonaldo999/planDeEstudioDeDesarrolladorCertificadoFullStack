¿Qué hace este simulador?

Este proyecto simula el flujo de asistentes a un festival de música. Imagina que tienes 4 puertas (Norte, Este, Sur, Oeste) por donde la gente entra.

1

Llegan personas a cada puerta

→
2

La puerta procesa según su capacidad

→
3

Si se llena, se deriva a la siguiente puerta

Conceptos Clave

  • Tick: Un momento específico en el tiempo
  • Capacity: Cantidad de personas que caben en una puerta
  • Overflow: Personas que no caben y deben derivarse
  • Queue: Cola de personas esperando

Estructura de Datos

Array de Gates (Puertas)

const morningGates = [
  { id: "North", capacity: 5, queue: [3, 6, 2, 4] },
  { id: "East", capacity: 3, queue: [2, 4, 3, 5] },
  { id: "South", capacity: 4, queue: [1, 2, 3, 1] },
  { id: "West", capacity: 2, queue: [4, 1, 2, 3] }
];

id: Nombre de la puerta

capacity: Máxima gente que puede pasar por tick

queue: Array con personas que llegan en cada tick

Tipos de Bucles en JavaScript

1. Bucle for...of

Se usa para recorrer colecciones (arrays) cuando sabes cuántos elementos hay.

for (const gate of gates) {
  summary[gate.id] = 0;
}
Usos principales:
  • Inicializar el resumen de throughput
  • Recorrer todas las puertas en cada tick

2. Bucle while

Se usa cuando no sabemos cuántas veces se repetirá. Se ejecuta mientras la condición sea verdadera.

while (currentTickQueue > 0 && processed < gate.capacity) {
  currentTickQueue--;
  processed++;
}
Usos principales:
  • Procesar personas una por una hasta llenar la puerta
  • Controlar los ticks de la simulación

3. Bucle for...in

Se usa para recorrer las propiedades de un objeto.

for (const gate in summary) {
  console.log(gate + ": " + summary[gate]);
}
Usos principales:
  • Mostrar el resumen final
  • Iterar sobre las claves de un objeto

4. Operador Módulo (%) - Lógica Circular

Para volver al inicio cuando llegas al final (como un círculo).

const nextGateIndex = (currentIndex + 1) % gates.length;
// Si estás en la última puerta, vuelves a la primera

Esto crea el efecto de "cola circular" para derivar personas.

Código Completo

Explicación por Pasos

  1. initializeThroughput: Crea el objeto para contar procesados
  2. processGateFlow: Procesa gente en una puerta (lógica core)
  3. rerouteOverflow: Deriva el exceso a la siguiente puerta
  4. handleGateAtTick: Controla todo lo que pasa en una puerta en un tick
  5. printSummary: Muestra el resultado final
  6. simulateFestival: Ejecuta la simulación completa
