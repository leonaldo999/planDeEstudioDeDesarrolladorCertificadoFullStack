REPASO

🚀 Introducción

En esta sección, repasaremos los conceptos básicos sobre los objetos en JavaScript, incluyendo su definición, cómo acceder a sus propiedades y algunos métodos útiles para trabajar con ellos.

Tema 1: Conceptos básicos sobre los objetos

  • Definición: Un objeto es una estructura de datos compuesta por propiedades. Una propiedad consta de una clave y un valor. Para acceder a los datos de un objeto, se puede utilizar la notación de puntos o la notación entre corchetes.
12345678 
const person = {
  name: "Alice",
  age: 30,
  city: "New York"
};

console.log(person.name);  // Alice
console.log(person["name"]); // Alice

Mapeo de Lectura en Memoria

Estructura interna de `person`:
🔑 "name" ➔ 📄 "Alice"
🔑 "age" ➔ 🔢 30
🔑 "city" ➔ 📄 "New York"
CONSOLE:
> "Alice"
> "Alice"

💡 Caso de uso Full-Stack: Usa obligatoriamente los corchetes [] si el nombre de tu propiedad tiene espacios en blanco (ej. person["full name"]) o si estás iterando llaves dinámicas que vienen de una consulta a una base de datos.

Para establecer una propiedad de un objeto existente, puedes utilizar tanto la notación de puntos como la notación entre corchetes junto con el operador de asignación.

12345678 
const person = {
  name: "Alice",
  age: 30
};

person.job = "Engineer";
person["hobby"] = "Knitting";
console.log(person);  // {name: 'Alice', age: 30, job: 'Engineer', hobby: 'Knitting'}

Expansión Dinámica del Objeto

Estructura final en memoria (Heap):
• name: "Alice" (Original)
• age: 30 (Original)
• job: "Engineer" ➔ Añadido via .
• hobby: "Knitting" ➔ Añadido via []
CONSOLE:
> Object { name: "Alice", age: 30, job: "Engineer", hobby: "Knitting" }

⚠️ Dato Full-Stack sobre const: Declarar el objeto con const impide que reasignes la variable a un objeto totalmente nuevo (por ejemplo: person = {} daría un error), pero **sí permite** añadir, modificar o eliminar sus propiedades internas sin ningún inconveniente.

Tema 2: Eliminación de propiedades de un objeto

  • Operador delete: Este operador se utiliza para eliminar una propiedad de un objeto.
12345678 
const person = {
  name: "Alice",
  age: 30,
  job: "Engineer"
};

delete person.job;

console.log(person.job); // undefined

Estado de las Claves Post-Eliminación

Estructura de llaves en ejecución:
✔ person.name ➔ "Alice"
✔ person.age ➔ 30
❌ person.job ➔ Removido de la estructura
CONSOLE:
> undefined

💡 Tip Full-Stack: Si quieres verificar de manera estricta si una propiedad fue completamente removida de un objeto en lugar de solo tener un valor vacío, puedes usar el operador binario in (por ejemplo: "job" in person), el cual devolverá false tras haber aplicado un delete.

Tema 3: Comprobar si un objeto tiene una propiedad

  • Método hasOwnProperty(): Este método devuelve un valor booleano que indica si el objeto tiene la propiedad especificada como una de sus propias propiedades.
1234567 
const person = {
  name: "Alice",
  age: 30
};

console.log(person.hasOwnProperty("name")); // true
console.log(person.hasOwnProperty("job")); // false

Estrategia de Inspección de Memoria

Rastreo del Método en `person`:
🔍 Buscando "name" ➔ ✨ ¡Encontrado en el primer nivel! ➔ true
🔍 Buscando "job" ➔ ❌ No existe dentro de las llaves ➔ false
CONSOLE:
> true
> false

⚡ Dato Full-Stack: En versiones recientes de ECMAScript, es una buena práctica y mucho más seguro usar Object.hasOwn(person, "name") en lugar de person.hasOwnProperty("name"), ya que evita errores raros si el objeto que estás evaluando fue creado sin prototipo (por ejemplo, con Object.create(null)).

  • Método Object.hasOwn(): Esta es la forma moderna y recomendada de comprobar si un objeto tiene una propiedad propia (no heredada). La sintaxis es Object.hasOwn(objeto, nombreDePropiedad). Devuelve true si la propiedad existe en el objeto y false si no existe, independientemente del valor de la propiedad. Esto lo hace más seguro que hasOwnProperty() y más confiable que if (obj.prop) cuando los valores pueden ser 0, false, null o undefined.
1234567 
const person = {
  name: "Alice",
  age: 30
;

console.log(Object.hasOwn(person, "name"); // true
console.log(Object.hasOwn(person, "job"); // false

Evaluación Estática de Seguridad

Mecanismo de Verificación Segura:
🛡️ Object.hasOwn(person, "name") ➔ El motor busca la llave en el mapa interno del objeto ➔ true
🛡️ Object.hasOwn(person, "job") ➔ La llave no existe en los registros de la estructura ➔ false
CONSOLE:
> true
> false

🔥 Por qué es más seguro: Si creases un objeto sin herencia con const obj = Object.create(null); obj.name = "Bob";, escribir obj.hasOwnProperty("name") rompería la aplicación con un error porque obj no heredó los métodos de Object. En cambio, Object.hasOwn(obj, "name") funciona perfectamente.

A continuación se muestra un ejemplo más detallado que ilustra por qué Object.hasOwn() es más confiable que comprobar el valor directamente:

123456789101112131415161718192021 
const settings = {
  darkMode: false,
  fontSize: 0,
  language: null
};

// Object.hasOwn() correctly sees these properties exist
console.log(Object.hasOwn(settings, "darkMode")); // true (value is false, but exists)
console.log(Object.hasOwn(settings, "fontSize")); // true (value is 0, but exists)
console.log(Object.hasOwn(settings, "theme"));    // false (property was never added)

// Using if() directly is unsafe for falsy values!
if (settings.darkMode) {
  console.log("Dark mode on"); // Does NOT print — misleading!
}

// Object.hasOwn() is the safe way
if (Object.hasOwn(settings, "darkMode")) {
  console.log("darkMode exists, value is:", settings.darkMode); // darkMode exists, value is: false
}

Análisis de Comportamiento Lógico

Evaluación de la Línea 13 (Insegura):
• Entrada: settings.darkMode ➔ Valor: false
• Comprobación implícita: Boolean(false) ➔ Coerción: falsy
• Resultado: El bloque `if` se salta por completo. (El sistema cree falsamente que la propiedad no existe o no está activa).
CONSOLE:
> true
> true
> false
> "darkMode exists, value is: false"

🛠️ Patrón Full-Stack Alternativo: Si deseas verificar existencia y además asignar un valor por defecto en una sola línea protegiéndote de los valores falsy, recuerda que puedes combinar la desestructuración con el operador de asignación condicional o el operador de fusión nula (Nullish Coalescing) como: const darkMode = settings.darkMode ?? true;.

  • Operador in: Este operador devolverá true si la propiedad existe en el objeto.
123456 
const person = {
  name: "Bob",
  age: 25
};

console.log("name" in person);  // true

Rastreo de Herencia e Inspección

Comportamiento del Operador:
• Evaluación: "name" in person
• Nivel 1 (Directo): Llave "name" encontrada ➔ true
• Nota de alcance: Si buscaras "toString" in person, también devolvería true debido a la herencia del prototipo.
CONSOLE:
> true

⚠️ Diferencia clave Full-Stack: Usa Object.hasOwn() si necesitas asegurarte de que la propiedad fue escrita explícitamente en el objeto (por ejemplo, datos del usuario). Usa el operador in si te interesa saber si la propiedad o el método están disponibles para ser usados, sin importar si provienen de la herencia o de la clase padre.

Tema 4: Acceso a propiedades de objetos anidados

  • Acceso a datos: Para acceder a las propiedades de objetos anidados se utiliza la notación de puntos o la notación entre corchetes, de forma muy similar a como se accede a las propiedades de objetos simples. Sin embargo, será necesario encadenar estos accesores para profundizar en la estructura anidada.
12345678910111213 
const person = {
  name: "Alice",
  age: 30,
  contact: {
    email: "alice@example.com",
    phone: {
      home: "123-456-7890",
      work: "098-765-4321"
    }
  }
};

console.log(person.contact.phone.work); // "098-765-4321"

Árbol de Evaluación de Caminos (Path Lookup)

Resolución de Capas en Memoria:
📦 Raíz: person
└── 🗂️ Capa 1: .contact ➔ { email, phone }
    └── 🗂️ Capa 2: .phone ➔ { home, work }
        └── 📄 Capa 3: .work ➔ "098-765-4321"
CONSOLE:
> "098-765-4321"

🧠 Tip de flexibilidad: Recuerda que puedes mezclar sintaxis según tus necesidades. La línea de impresión también podría haberse escrito dinámicamente como person["contact"].phone["work"] o usando variables para definir las llaves en tiempo de ejecución.

Tema 5: Tipos de datos primitivos y no primitivos

  • Tipos de datos primitivos: Estos tipos de datos incluyen números, cadenas, valores booleanos, null, undefined y símbolos. Se denominan primitivos porque representan valores únicos y no son objetos. Los valores primitivos son inmutables, lo que significa que, una vez creados, su valor no se puede modificar.
  • Tipos de datos no primitivos: en JavaScript, estos son objetos, que incluyen objetos regulares, arrays y funciones. A diferencia de los primitivos, los tipos no primitivos pueden contener múltiples valores como propiedades o elementos.
123456789 
// Primitivos: Inmutables y guardados por valor
let name = "Alice";
name.toUpperCase(); 
console.log(name); // "Alice" (El valor original no cambia)

// No Primitivos: Mutables y guardados por referencia
const person = { name: "Alice" };
person.age = 30; // Añade múltiples valores dinámicamente
console.log(person); // { name: "Alice", age: 30 }

Estructura de Datos en Ejecución

Mapa de Clasificación de JavaScript:
🔹 Primitivos (Valor único): Number, String, Boolean, null, undefined, Symbol.
🔸 No Primitivos (Contenedores): Object ➔ {} (Objetos), [] (Arrays), function() (Funciones).
CONSOLE:
> "Alice"
> Object { name: "Alice", age: 30 }

⚠️ Efecto secundario Full-Stack: Debido a que los objetos se guardan por referencia, hacer const copia = person; copia.name = "Bob"; modificará automáticamente el nombre dentro de person. Para duplicar un objeto sin mantener este enlace, debes clonarlo usando el operador spread: const copia = { ...person };.

Tema 6: Métodos de objeto

  • Definición: Los métodos de objeto son funciones asociadas a un objeto. Se definen como propiedades de un objeto y pueden acceder a los datos del objeto y manipularlos. La palabra clave this dentro del método hace referencia al propio objeto, lo que permite acceder a sus propiedades.
123456789 
const person = {
  name: "Bob",
  age: 30,
  sayHello: function() {
    return "Hello, my name is " + this.name;
  }
};

console.log(person.sayHello()); // "Hello, my name is Bob"

Resolución del Entorno de Ejecución (Scope)

Enlace dinámico de `this`:
• Invocación ➔ person.sayHello()
• Evaluación ➔ El punto vincula la función al objeto izquierdo.
• Contexto interno ➔ this muta hacia ➔ Objeto person
• Extracción ➔ this.name se resuelve como ➔ "Bob"
CONSOLE:
> "Hello, my name is Bob"

⚠️ Peligro con Arrow Functions: Evita declarar métodos de objeto usando funciones flecha (ej. sayHello: () => this.name). Las funciones flecha no crean su propio contexto this, sino que lo heredan del ámbito global externo, lo que causaría que this.name devuelva undefined.

Tema 7: Constructor de objetos

  • Definición: En JavaScript, un constructor es un tipo especial de función que se utiliza para crear e inicializar objetos. Se invoca con la palabra clave new y permite inicializar propiedades y métodos en el objeto recién creado. El constructor Object() crea un nuevo objeto vacío. 
            
              new Object()

Tema 8: Uso del operador de encadenamiento opcional (?.)

  • Definición: Este operador te permite acceder de forma segura a las propiedades de un objeto o llamar a métodos sin tener que preocuparte de si existen o no.
12345678910111213 
const user = {
  name: "John",
  profile: {
    email: "john@example.com",
    address: {
      street: "123 Main St",
      city: "Somewhere"
    }
  }
};

console.log(user.profile?.address?.street); // "123 Main St"
console.log(user.profile?.phone?.number);   // undefined

Simulación del Cortocircuito del Motor

Rastreo de la Línea 13:
1. user.profile ➔ Existe (Objeto). Continúa...
2. ?.phone ➔ Se evalúa el subobjeto ➔ undefined
3. Activación de Cortocircuito ➔ El operador detecta la ausencia, cancela el intento de leer .number de inmediato y retorna ➔ undefined
CONSOLE:
> "123 Main St"
> undefined

💡 Combinación de Oro en Full-Stack: En producción, es sumamente habitual combinar el encadenamiento opcional con el operador de fusión nula (Nullish Coalescing `??`) para definir un valor de respaldo amigable en caso de ausencia: const phone = user.profile?.phone?.number ?? "Sin teléfono registrado";.

Tema 9: Descomposición de objetos

  • Definición: La descomposición de objetos permite extraer valores de los objetos y asignarlos a variables de una manera más concisa y legible.
123456 
const person = { name: "Alice", age: 30, city: "New York" };

const { name, age } = person;

console.log(name); // Alice
console.log(age);  // 30

Mapeo de Variables Desestructuradas

Extracción Selectiva en Línea 3:
• Entrada: Objeto person con 3 propiedades.
• Extracción 1 ➔ Busca la clave "name" ➔ Crea const name = "Alice".
• Extracción 2 ➔ Busca la clave "age" ➔ Crea const age = 30.
• Ignorado ➔ Propiedad city permanece intacta dentro del objeto, pero no se crea una variable para ella.
CONSOLE:
> "Alice"
> 30

💡 Tip avanzado de sintaxis: Si necesitas que las variables finales tengan nombres diferentes a las llaves originales del objeto, puedes renombrarlas en la misma línea usando los dos puntos, por ejemplo: const { name: firstName, age: userAge } = person;.

Tema 10: Trabajar con JSON

  • Definición: JSON son las siglas de JavaScript Object Notation (JavaScript Object Notation). Se trata de un formato de datos ligero y basado en texto que se utiliza habitualmente para intercambiar datos entre un servidor y una aplicación web.
123456 
{
  "name": "Alice",
  "age": 30,
  "isStudent": false,
  "list of courses": ["Mathematics", "Physics", "Computer Science"]
}

Intercambio de Datos en la Red

Ciclo de vida JSON en JavaScript:
⚡ De Objeto a Texto (Envío):
JSON.stringify(objetoJS) ➔ Convierte a la cadena de texto JSON mostrada a la izquierda para poder viajar por HTTP.

⚡ De Texto a Objeto (Recepción):
JSON.parse(textoJSON) ➔ Transforma este bloque de texto plano recibido del servidor de vuelta en un objeto interactivo programable.
VALIDATOR STATUS:
> JSON estructurado correctamente (Valid 100%)

💡 Caso de uso real: Como las claves en JSON siempre son cadenas envueltas en comillas, puedes usar nombres de propiedades con espacios sin problemas (como "list of courses"). Al parsearlo en JavaScript, podrás acceder a él mediante la notación de corchetes: data["list of courses"].

  • JSON.stringify(): Este método se utiliza para convertir un objeto de JavaScript en un string JSON. Resulta útil cuando se desea almacenar o transmitir datos en un formato que se pueda compartir o transferir fácilmente entre sistemas.
12345678 
const user = {
  name: "John",
  age: 30,
  isAdmin: true
;

const jsonString = JSON.stringify(user;
console.log(jsonString; // '{"name":"John","age":30,"isAdmin":true}'

Efecto de la Serialización de Datos

Transformación del tipo de dato:
• Entrada: Variable user ➔ Tipo: Object (Estructura viva)
• Salida: Variable jsonString ➔ Tipo: String (Texto plano plano)
• Cambio de formato: Las llaves sin comillas como name: se transforman estrictamente en claves con comillas dobles "name":.
CONSOLE:
> '{"name":"John","age":30,"isAdmin":true}'

⚠️ Comportamiento con funciones y undefined: Ten en cuenta que JSON.stringify() omitirá automáticamente cualquier propiedad del objeto cuyo valor sea una función o sea undefined, ya que esos tipos de datos no existen ni son válidos dentro de la especificación estándar de JSON.

  • JSON.parse(): Este método convierte un string JSON de nuevo en un objeto JavaScript. Esto resulta útil cuando recuperas datos JSON de un servidor web o de localStorage y necesitas manipularlos en tu aplicación.
123456 
const jsonString = '{"name":"John","age":30,"isAdmin":true}';
const userObject = JSON.parse(jsonString;

// result: { name: 'John', age: 30, isAdmin: true }
console.log(userObject;

Reconstrucción Estructural en RAM

Efecto de la Deserialización:
• Entrada: jsonString ➔ Tipo: String (Texto inerte)
• Salida: userObject ➔ Tipo: Object (Estructura programable)
• Operaciones permitidas post-parse: Ya puedes hacer userObject.name o alterar sus valores en caliente.
CONSOLE:
> Object { name: "John", age: 30, isAdmin: true }

⚠️ Blindaje Full-Stack obligatorio: Si el string de origen tiene un error tipográfico o una coma de más, JSON.parse() romperá tu aplicación lanzando un error fatal (SyntaxError). En entornos reales de producción, envuelve siempre este método dentro de un bloque de seguridad try { ... } catch (e) { ... }.
